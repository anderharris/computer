Cryptography

What is Cryptography?

According to Kaspersky, a world leader in Computer Security, Cryptography can be defined as the study of secure communication techniques - allowing only authorised recipients to view its content.

Although historically the field primarily focused on linguistics and patterns in lexicon, the emergence of the digital age has shifted the definition into one which strongly emphasises Information Theory, Number Theory, Combinatorics, Statistics, and other areas of Mathematics. Cryptography is so vital today since data is routed between computers all across the world, and you wouldn't want the wrong people seeing your private information. This Article aims to give you a glimpse into the wonderful world of Cryptography in Computer Science.

Symmetric Encryption

Symmetric Encryption uses the same key to both encrypt and decrypt data. A vital benefit to Symmetric Encryption is its speed - since the same key is used for both the sender and recipient, time isn't wasted generating multiple keys; plus, keys for Symmetric Encryption are often shorter, meaning the encryption and decryption process takes less time.

Asymmetric Encryption

Public-Key Encryption, or Asymmetric Encryption is a form of encryption where a different key is used for encrypting and decrypting data. There will be two separate keys: A Public Key and a Private Key. The Public Key will be freely available and can be used by anyone to encrypt data. However, due to some cunning maths, the same key cannot be used to decrypt the data - only the the Private Key can do this.

The algorithm commonly used to generate these keys is called RSA, and makes use of Euler's Totient Function to prevent the Private Key from being figured out from the Public Key (meaning only the intended recipient will only ever be able to read the private message).

For a more in-depth explanation, watch this lecture by Ruhr University Bochum.

The main attraction to Asymmetric Encrpytion is its security. Since a different key is used for encryption and decryption, there is no to transfer the key used for decryption - only the Public Key needs to be shared to the sender!

Asymmetric Encryption is used in:

  • Key Exchange (see below)
  • Email Security
  • Web Security
